Moneger

A privacy-first, local-first money management app built so your financial source of truth stays on your device.

Open Web App Download / Install App
Storage model IndexedDB first
Remote role Auth + optional encrypted sync
Core principle Your Money. Your Control.
Why I built this

Privacy should be the default for personal finance

Most money apps normalize storing sensitive records on a remote server. Moneger starts from the opposite assumption: users should not lose access to their own financial records because a server is down or a sync service fails.

How I built this

Local-first architecture with a modern web stack

Moneger uses Next.js, TypeScript, Tailwind CSS, Dexie, Chart.js, and Firebase Auth. IndexedDB is the primary ledger database. Firebase is kept limited to auth and optional encrypted sync scaffolding.

How to use this

Track income, expenses, debts, owed money, and banks

Create an account, enter records, view dashboard totals and charts, check settlement reminders, and keep your records available offline. If you later want sync, add encrypted sync carefully rather than pushing raw finance data upstream.

Feature Summary

Included

  • Signup, login, password reset, and sign out flow
  • Income, expense, debt, owed, and bank tracking
  • Dashboard charts and mixed-currency totals
  • Exchange-rate cache with offline fallback
  • Notification and encrypted sync scaffolding

Principles

  • Local-first source of truth
  • Offline-first core experience
  • No plaintext remote finance storage by default
  • Reusable architecture for future mobile adaptation
